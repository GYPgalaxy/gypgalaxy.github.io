Redis简介

Redis（全称为Remote Dictionary Server）是一个开源的高性能键值对存储系统，具有快速、灵活和可扩展的特性。它是一个基于内存的数据结构存储系统，可以用作数据库、缓存和消息代理。
使用场景： Redis 最常使用的场景就是缓存了，但是 Redis 不仅仅可用作缓存，还可以使用 Redis 单线程的特性用作计数器; 使用 Redis 发布订阅的特性用作消息队列系统; 使用 Redis 有序集合的数据结构完成排行榜功能; 使用 Redis 分布式锁功能；Redis 与社交网络结合；使用 Redis 布隆过滤器完成限流功能等。很多的场景都可以使用 Redis。

分布式数据库CAP原理

传统的关系型数据库四要素：ACID->原子性、一致性、隔离性、持久性

CAP是什么：

  • C(Consistency) 强一致性。 一致性要求系统的数据在任何时刻都保持一致的状态。也就是说，如果一个数据副本被修改，那么所有副本都必须被相应地更新，以保持数据的一致性。在分布式系统中，一致性的实现可以通过各种机制，例如复制、同步和协调来实现。
  • A(Availability) 高可用性。 可用性要求系统在面对用户请求时能够提供服务，并且能够在合理的时间内响应这些请求。换句话说，系统不能因为某个节点或组件的故障而完全停止对外提供服务。
  • P(Partition tolerance)分区容错性。 分区容错性是指系统能够在面对网络分区（节点之间无法相互通信）的情况下继续运行。在分布式系统中，网络分区是不可避免的，因为系统的各个节点之间通过网络进行通信，而网络可能会发生延迟、丢失消息或者分区等问题。

分布式数据库CAP特性不能三个都满足，只能满足其中的两条，其中P即分区容错性必须要实现。
传统的Oracle Mysql等满足CA
AP+弱一致性 :大多数网站架构的选择
CP :Redis Mongodb
为什么只能满足两个：一致性要求系统的数据在任何时刻都保持一致的状态，而可用性要求系统能够在合理的时间内响应用户请求。当系统面临网络分区时，数据副本之间的同步可能会受到延迟或中断，这就导致了一致性和可用性之间的冲突。如果系统选择保证一致性，那么在网络分区期间，系统可能无法提供服务，因为需要等待分区解除以保持数据的一致性。相反，如果系统选择保证可用性，那么在网络分区期间，系统可能会继续提供服务，但数据副本之间的一致性可能会受到影响。

BASE是什么

BASE是为了解决关系数据库强一致性引起的问题而引发的可用性降低而提出的解决方案。

  • BA(Basically Available)基本可用。 BASE 强调系统在面对故障或分区时，仍然能够保持基本的可用性，即系统能够继续提供有限的服务，而不会完全停止响应。这与传统的强一致性模型（如 ACID）不同，它允许系统在某些情况下放宽一致性要求，以换取更高的可用性。
  • S(Soft state)软状态。 软状态指的是系统中的数据状态可以随时变化，而不需要遵循强一致性约束。在 BASE 模型中，系统允许数据处于中间状态或部分更新状态，而不要求立即达到全局一致状态。这使得系统能够更快地响应用户请求和处理大量的并发操作。
  • E(Eventually consistent) 最终一致性。最终一致性是指系统中的所有数据副本最终会达到一致的状态，但在数据更新过程中可能出现一段时间的不一致。在 BASE 模型中，系统允许数据副本在分布式环境中存在延迟和异步复制的情况，通过一定的机制最终将数据副本调整为一致状态。

常用指令

1. 通用

序号 命令 描述 时间复杂度
1 keys [pattern] 列出系统中所有的 key O(n)
2 dbsize Redis 系统所有 key 的个数 O(1)
3 exists key Redis 系统是否存在该 key，存在返回1，否则返回0 O(1)
4 del key… 删除 Redis 中的 key，删除不存在的key返回0，否则返回key个数 O(1)
5 expire key seconds 设置 key 的过期时间，单位是秒，pexpire 单位是毫秒 O(1)
6 ttl key 显示 key 剩余的过期时间，返回 -1 表示没有设置过期时间，-2表示key已删除，pttl，单位是毫秒 O(1)
7 persist key 去掉 key 的过期时间 O(1)
8 type key 查看 key 的数据类型 O(1)
9 randomkey 随机返回系统中存在的key
10 rename key1 newKey 给 key 重命名
11 renamenx key1 newKey 给 key 重命名，如果newKey已存在则重命名失败，返回0，否则返回1
12 move key DBNO 将指定的key移动到指定的数据库，redis默认16个库，默认使用 0号库
13 select DBNO 使用 redis 的某个数据库，类似于 mysql 的 use dbname
